## Today I Learned

### Formal Verification of Protocols (Process Calculus Applied to MCP)
**Why this came up:** ArXiv paper applying process calculus to formally verify MCP — the protocol that all AI agents use to connect to tools.

Process calculus is a mathematical framework for reasoning about systems that communicate. It treats each party in a protocol (the AI model, the tool server) as a "process" and their interactions as "message exchanges," then uses mathematical rules to prove properties about those exchanges — like whether messages can be lost, whether the system can deadlock (both parties waiting forever for the other to respond), or whether two protocols are equivalent.

The paper proved that MCP and Schema-Guided Dialogue (SGD, Google's older protocol) are "bisimilar" — their communication patterns match structurally. But the reverse mapping (going from SGD to MCP) is "partial and lossy," meaning some things SGD can express, MCP cannot. This matters because it identifies specific categories of agent-tool interactions that can silently fail under MCP — the protocol doesn't have the type expressivity to represent certain tool schemas correctly.

The proposed fix (MCP+) adds five type-system extensions: union types, intersection types, conditional types, recursive types, and dependent types. These are all concepts from type theory (the mathematical study of how to classify and validate data). Each extension closes a specific gap in what tool schemas MCP can represent.

### Quick Glossary
| Term | Meaning |
|------|---------|
| Process calculus | Mathematical framework for analyzing communicating systems |
| Bisimilar | Two systems that match in their observable communication behavior |
| Type system | Rules that classify and constrain what kinds of data can flow through a system |
| Deadlock | State where all parties are waiting for each other and nothing can proceed |
| Union type | A value that can be one of several types (e.g., "string OR number") |
| Dependent type | A type whose definition depends on a value (e.g., "array of length N" where N is computed) |
