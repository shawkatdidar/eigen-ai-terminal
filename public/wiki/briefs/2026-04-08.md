## Today I Learned

### Test-Time Training — Making Models Learn While They Think
**Why this came up:** In-Place TTT paper accepted as ICLR 2026 Oral

Current LLMs have **static weights** — once training finishes, the model's internal parameters are frozen. Every query uses the same fixed model, regardless of what's in the context. Context windows let models see more text, but seeing text isn't the same as *learning from it*. A model with a 1M token context can reference information far back in the conversation, but its internal knowledge (encoded in weights) doesn't update.

**Test-Time Training (TTT)** is a technique that lets models update a small subset of their parameters *during inference* — as they process your query, not during the training phase. The In-Place TTT paper makes this practical by identifying which parameters to update: the final projection matrix in each MLP block (MLP = multi-layer perceptron, the feedforward sub-network inside each transformer layer that processes each token's representation after attention). By treating these specific matrices as "fast weights" (weights that change quickly at runtime, as opposed to the "slow weights" that were set during training), the model can adapt to new patterns in real-time.

Why this matters for evaluation: The distinction between "seeing context" and "learning from context" is critical. A model that *learns* from a long document can extract structural patterns that a model merely *referencing* the document cannot. This is especially relevant for tasks like codebase understanding (learning the patterns of a specific codebase, not just seeing the files) and personalization (adapting to a user's style over time).

### Quick Glossary
| Term | Meaning |
|------|---------|
| Test-Time Training (TTT) | Updating model parameters during inference, not just during training |
| Fast weights | Parameters that change at runtime to adapt to current input |
| MLP (Multi-Layer Perceptron) | Feedforward sub-network in each transformer layer; processes token representations after attention |
| ICLR | International Conference on Learning Representations — top ML venue |
| Polynomial Mixer | Architecture that replaces quadratic-cost attention with polynomial functions for linear O(n) complexity |
