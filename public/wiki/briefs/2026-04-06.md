## Today I Learned

### The Data Processing Inequality and Why It Challenges Multi-Agent AI

**Why this came up:** arXiv:2604.02460 used DPI to argue single-agent outperforms multi-agent when compute is equalized.

The **Data Processing Inequality (DPI)** is a fundamental theorem from information theory: if you have a chain of processing steps X → Y → Z, each step can only *lose* information, never gain it. The mutual information I(X;Z) ≤ I(X;Y). In plain terms: every time data passes through a processing layer, some information gets degraded.

Applied to multi-agent AI systems: when Agent A sends a summary to Agent B, information about the original problem is *necessarily* lost in that summary. The more agents in the chain, the more information lost. A single agent with the same total compute budget sees all the original information at once.

The paper tested this across three model families and found that multi-agent gains vanish when you give the single agent the same number of total reasoning tokens. The perceived benefit of multi-agent systems in benchmarks comes from accidentally giving the multi-agent setup *more total compute* (each agent gets its own token budget).

**Why this matters for judgment:** The multi-agent trend (Cursor 3 Agents Window, oh-my-codex, CrewAI, AutoGen) is one of the hottest areas in AI tooling. This paper provides a principled reason to be skeptical — not "multi-agent doesn't work" but "multi-agent is less efficient per token than single-agent, so the benefit must come from something other than accuracy" (specialization, parallelism, fault tolerance). When evaluating multi-agent products, ask: *does this system add value through coordination benefits that overcome the DPI information loss, or is it just burning more tokens?*

### Quick Glossary

| Term | Meaning |
|------|---------|
| Data Processing Inequality (DPI) | Information theory theorem: processing steps can only lose information, never create it |
| Agentic GRPO | Group Relative Policy Optimization adapted for multi-stage agent rollouts with delayed rewards |
| KV cache | Key-Value cache — stores representations of previous tokens during inference to avoid recomputation |
| ASR (Attack Success Rate) | Percentage of adversarial attempts that successfully bypass model safety guardrails |
| MoE (Mixture of Experts) | Architecture where only a fraction of model parameters activate per query, reducing compute |
| Constitutional AI | Training approach where the model generates its own guidelines and uses them for self-improvement |
