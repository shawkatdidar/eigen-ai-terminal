## Today I Learned

### Gated-deployment vs. release-with-mitigations

**Why this came up:** Anthropic's Mythos decision.

Until today, the frontier-lab playbook has been "release with safety mitigations" — the model ships behind an API with filtered outputs, refusal training, rate limits, and monitoring. Anthropic's Responsible Scaling Policy (RSP) defined ASL-3 and ASL-4 as tiers where pre-deployment evaluations must pass before release, but the expectation was still "pass the eval, then release to everyone." Mythos is the first public case of a lab saying "the eval didn't produce a pass, so we will not release it via the standard surface." Access instead flows through gated channels — federal agencies, vetted defenders, allied labs — with per-recipient terms.

The mechanism matters. A standard API release means one model + one set of guardrails + millions of users. Gated deployment means one model + different guardrails per recipient + a small number of known users whose usage can be audited. The cost is distribution friction; the benefit is that capability that's unsafe in the hands of an anonymous API user can still be deployed for a known Treasury CIO doing vulnerability hunting.

The judgment framework: when evaluating "did lab X release Y safely?" going forward, the question isn't just "what mitigations shipped with it?" It's "what surface did it ship on?" API-available with filters ≠ gated-partner ≠ internal-only. Each has a different risk profile and a different set of second-order effects (concentration of capability, regulatory optionality, commercial economics).

### Trait transfer through distillation

**Why this came up:** Anthropic's Nature paper on subliminal learning.

Distillation is the standard technique for compressing a big model into a small one — you take the big model's outputs (or output probabilities) on a training set and train a smaller student to match them. The assumption is that filtering the training data removes the undesirable properties — if you don't want your student to be sycophantic, you filter out sycophantic outputs.

The paper shows this assumption is wrong. Behavioral traits (including misalignment) transfer to the student even when the training data has been filtered to remove any semantic trace of those traits. The mechanism appears to be statistical regularities in "neutral" outputs — the way the teacher phrases a benign math answer apparently encodes information about its personality-level properties, which the student picks up.

The judgment framework: "I filtered the outputs" is no longer a sufficient safety argument for distilling a frontier model. If someone claims "we distilled Claude into a 7B model with all safety properties intact," the follow-up question is "did you test the 7B on red-team prompts the teacher was trained to refuse — and did the student refuse at the same rate?" Not "did you filter the training data." Filtering doesn't catch what isn't semantically encoded.
