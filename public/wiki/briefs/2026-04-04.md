## Today I Learned

### AI-Powered Vulnerability Discovery at Scale
**Why this came up:** Claude Code found a 23-year-old Linux kernel vulnerability, and the MAD Bugs campaign found 500+ zero-days.

Traditional vulnerability discovery relies on human security researchers manually reading source code, fuzzing (automated random input testing), and static analysis tools that check for known patterns. These methods miss bugs because humans can't read millions of lines of code, fuzzers only test execution paths they happen to hit, and static analyzers only catch patterns they're programmed to recognize.

What Claude Code did differently: it read the NFSv4 driver source code with full context (the specification says the lock owner field can be up to 1024 bytes, but the server only allocated 112 bytes). This is a semantic understanding gap — the bug exists in the mismatch between the protocol spec and the implementation. No fuzzer would catch this unless it happened to generate a >112 byte lock owner. No static analyzer checks protocol spec compliance. But an LLM that has read both the spec and the code can reason about the mismatch.

The scale result (500+ zero-days in a month) matters because it proves this isn't a lucky one-off — it's a systematic capability. The structural asymmetry: finding takes hours (LLM speed), patching takes days-weeks (human review, testing, deployment). Every hour an LLM runs, the known-vulnerability gap widens.

### Quick Glossary
| Term | Meaning |
|------|---------|
| Heap buffer overflow | Writing data past the end of allocated memory on the heap, potentially overwriting adjacent data structures or enabling code execution |
| Zero-day | A vulnerability unknown to the software vendor, meaning zero days of available patches |
| NFSv4 | Network File System version 4 — protocol for accessing files over a network, built into the Linux kernel |
| Fuzzing | Automated testing technique that feeds random/malformed inputs to find crashes or unexpected behavior |
| Static analysis | Examining code without executing it, checking for known vulnerability patterns |
