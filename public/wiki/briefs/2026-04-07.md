## Today I Learned

### Controlled-Release as a Model Distribution Paradigm
**Why this came up:** Anthropic's decision to withhold Mythos from general release and deploy it only through Project Glasswing

Until today, frontier AI models have been distributed along a simple spectrum: fully open-weight (Llama, Gemma, Mistral), API-access (Claude, GPT), or proprietary-internal (never released). Mythos introduces a fourth pattern: **consortium-controlled release** — the model exists, its capabilities are demonstrated publicly, but access is restricted to a vetted group of organizations for specific use cases.

This matters because it changes the dynamics of AI capability deployment. The traditional assumption has been that once a model is released (via API or weights), its capabilities are available to essentially anyone. Consortium release breaks this by allowing the creator to deploy capabilities selectively — in this case, channeling offensive cybersecurity capability exclusively toward defense.

The precedent cuts both ways. On one hand, it demonstrates responsible deployment of dual-use capabilities. On the other, it creates a capability gap where consortium members have access to tools that the broader ecosystem does not, potentially concentrating security advantages in large organizations already partnered with Anthropic.

### Quick Glossary
| Term | Meaning |
|------|---------|
| Zero-day vulnerability | A software flaw unknown to the vendor — no patch exists. Called "zero-day" because the vendor has had zero days to fix it. |
| SWE-bench Verified | A benchmark measuring an AI model's ability to resolve real GitHub issues. Higher = better at autonomous software engineering. |
| Adversarial distillation | The practice of systematically querying a large model's API to extract its behavior patterns, then using those outputs to train a smaller, cheaper model. The "student" model learns to approximate the "teacher" without access to the teacher's weights. |
| Dynamic Sparse Attention | An attention mechanism (the core computation in transformer models) where the number and pattern of attention connections adapts per input, rather than being fixed. Reduces compute for inputs that don't need full attention while preserving quality for complex inputs. |
| Consortium release | A distribution model where a capability is deployed only to a vetted group of organizations, rather than publicly via API or open weights. |
