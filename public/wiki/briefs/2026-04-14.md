## Today I Learned

### Diffusion Language Models vs Autoregressive Models
**Why this came up:** I-DLM paper matching autoregressive quality at 3x throughput

Every major production LLM (GPT-5.4, Claude Opus 4.6, Gemini 3.1) uses **autoregressive generation** — producing one token at a time, left to right. Each token depends on all previous tokens, which means generation is inherently sequential. You can't parallelize the output.

**Diffusion language models (DLMs)** take a different approach borrowed from image generation. Instead of generating tokens sequentially, they start with noise (random token positions) and iteratively refine all positions simultaneously. Each "denoising step" improves the entire output at once. This is inherently parallelizable — multiple tokens improve in a single forward pass.

The tradeoff has always been quality: DLMs produced worse text than autoregressive models at the same parameter count. I-DLM closes this gap using **Introspective Strided Decoding (ISD)** — during generation, the model simultaneously generates new tokens AND verifies previously generated tokens in the same forward pass. This self-verification step catches errors that would otherwise accumulate.

Why this matters: if DLMs can match AR quality at production scale, inference cost drops substantially for high-concurrency workloads (many users generating simultaneously), because the GPU can process more tokens per forward pass.

### The CLAUDE.md Paradox
**Why this came up:** Study of 679 rule files showing positive instructions hurt coding agents

When humans write instruction files for coding agents, we naturally write both "do this" and "don't do this" rules. The study found that only "don't do this" rules (negative constraints) actually improve agent performance. "Do follow the existing code style" actively degrades performance. And random rules help as much as expert-curated ones.

The likely mechanism: negative constraints narrow the action space (fewer wrong moves possible), while positive directives add conflicting optimization targets that interfere with the model's own learned behavior. The benefit of any rules — even random ones — comes from context priming (additional context helps the model ground its behavior), not from the specific content.

### Quick Glossary
| Term | Meaning |
|------|---------|
| Autoregressive (AR) | Generating output one token at a time, each conditioned on all previous tokens |
| Diffusion Language Model (DLM) | Generating output by iteratively denoising all tokens in parallel |
| Introspective Strided Decoding | Technique where generation and self-verification happen in the same forward pass |
| Negative constraint | A rule telling an agent what NOT to do ("do not refactor unrelated code") |
| Context priming | The phenomenon where additional context in the prompt helps ground model behavior, regardless of specific content |
| Neocloud | Newer GPU cloud providers (Fluidstack, CoreWeave, Lambda) competing with AWS/Azure/GCP for AI workloads |
