## Today I Learned

### Frequency-Domain Processing as an Alternative to Attention
**Why this came up:** Kathleen paper — 733K parameter model with no tokenizer or attention mechanism outperforming 16x larger models.

Standard transformer models process text through **attention mechanisms** — computing which tokens in a sequence are relevant to each other via dot-product similarity scores. This works but is computationally expensive (quadratic cost with sequence length, as you learned in Session 5).

Kathleen takes a fundamentally different approach: **frequency-domain processing**. Instead of computing pairwise token relationships, it converts input byte sequences into frequency representations using **RecurrentOscillatorBanks** — banks of damped sinusoidal oscillators (mathematical waves that gradually decay) that respond to different patterns in the input. Each oscillator resonates when it encounters a pattern it's tuned to detect, producing a frequency-domain representation of the text.

The key architectural components:
- **FFT-Rotate Wavetable Encoder**: Maps each of the 256 possible byte values to a position on a complex unit circle using just 256 floats — replacing the typical 65,536-parameter embedding table that transformer tokenizers require.
- **PhaseHarmonics**: A non-linear activation function (the component that introduces the ability to model complex relationships) with only 6 learnable parameters, using sinusoidal transformations.

Why this matters: the entire model has 733K parameters total — that's 0.0006% the size of a typical 1B-parameter model. It achieves 88.6% on IMDB sentiment classification (beating models with 16x more parameters) because frequency-domain processing can capture sequential patterns efficiently without the overhead of attention.

**Limitation:** Currently only demonstrated on classification tasks (deciding if text is positive/negative, which category a news article belongs to). Whether frequency-domain processing can match transformers on generation tasks (producing new text token by token) remains unproven. The paper is an EDGE-01 signal — interesting but far from displacing transformers.

### Quick Glossary
| Term | Meaning |
|------|---------|
| Frequency-domain processing | Representing signals as combinations of oscillating waves rather than as sequences of discrete values — the same math used in audio processing and image compression |
| RecurrentOscillatorBank | A bank of damped sinusoidal oscillators that respond to different input patterns — each oscillator "rings" when it detects its target pattern |
| FFT (Fast Fourier Transform) | Algorithm that converts a signal from time/sequence domain to frequency domain, revealing which periodic patterns are present |
| DCO (Developer Certificate of Origin) | Legal certification that a contributor has the right to submit code to an open-source project — the Linux kernel's foundation of contributor trust |
| CMORG | Cross Market Operational Resilience Group — BOE's forum for discussing operational risks to the UK financial system |
| PJM | PJM Interconnection LLC — operator of the electrical grid serving 13 US states and 65 million people |
